[Django 뷰 만들기]

지난 장에서 만들었던 버그를 잡을 시간이 왔어요.

뷰(view) 는 어플리케이션의 로직을 넣는 곳이에요. 뷰는 앞 챕터에서 만들었던 모델 에게서
필요한 정보를 받아와서 템플릿에 전달하는 역할을 합니다. 다음 장에서 템플릿을 만들어 볼 거에요.
뷰는 파이썬 들어가기 장에서 했던 것 보다 금 복잡해 보이지만, 그래도 결국 파이썬의 메소드일 뿐이에요.

뷰는 view.py 파일 안에 있습니다. 우리는 views를 
blog/views.py 파일 안에 추가할 거에요.


[blog/views.py]

좋아요 . 이제 파일을 열고 안에 있는 내용을 살펴봅시다.

from django.shortcuts import render

# Create your views here.

여기서는 별 내용이 없네요. 가장 간단한 view를 이렇게 만들어봅니다.

def post_list(request):
	return render(request, '/blog/post_list.html',{})

방금 우리는 post_list 라는 메서드를 만들었습니다.
( def 가 메소드를 만들 때 사용하는 키워드였죠?)
그리고 이 메서드는 요청(request)을 넘겨 받아 render 메소드를 호출합니다.
render 메서드는 넘겨진 요청(request)과 blog/post_list.html 템플릿 받아 리턴된 내용이 
브라우저에 보여지게 됩니다.

파일을 저장하고, http://127.0.0.1:8000/로 접속해 결과를 확인해보세요.
